# MovieHub
MovieHub - приложение для получения информации о фильмах и сериалах из разных открытых источников. Информация представляет собой агрегированые данные о запрашиваемом фильме/сериале, как в укороченном виде, так и в развернутом. 

## <img width="40" height="40" alt="compliance_15266941" src="https://github.com/user-attachments/assets/af535f87-881b-4ca1-8997-b7ca1ce3a427" />  Техническое задание
Цель проекта: агрегатор данных о фильмах и сериалах из нескольких внешних источников(Кинопоиск и TMDB).
Задачи:
1. Реализовать пулл api для получения информации о фильмах и сериалах:
   * GET /api/search?query=TitleMoovie - поиск фильма/сериала по названию и вывод краткой информации о нем
   * GET /api/movie/{id} - возвращает расширенные данные по запрашиваемому фильму/сериалу

    Источники данных:
    * TMDB 
    * KinoPoisk Unofficial API
  
    API источников:
    * TMDB  GET https://api.themoviedb.org/3/search/movie?query=Inception&api_key=YOUR_KEY (документация https://developer.themoviedb.org/reference/intro/getting-started)
    * KinoPoisk https://api.kinopoisk.dev/v1.4/movie/search (документация https://api.kinopoisk.dev/documentation#/)
  
2. Запросы к источникам выполнять одновременно. Полученную информацию необходимо распарсить для создаваемых api (1) и 
в ответе предоставить агрегированную информацию по фильмам/сериалам
3. Реализовать проверку кеша Redis. Сначала проверяем, есть ли нужный ответ в Redis, если есть, используем его, если нет
идем в БД (4).
4. Сохранять информацию о фильмах в БД.

5. Логировать все запросы в БД. Логируем дату запроса, 

Реализовать логику таким образом, что при запросе api сначала проверять информацию в БД, если она есть, выдавать пользователю эту информацию, если нет, то необходимо сделать запрос к источникам данных.
